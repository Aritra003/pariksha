# Contributing to Pariksha

Thanks for your interest in contributing.

## How to Contribute

1. **Fork** the repository and clone your fork
2. **Create a branch** from `main`: `git checkout -b feat/your-feature`
3. **Make your changes** — keep commits focused and atomic
4. **Commit** using conventional commit format:
   - `feat: add X` — new feature
   - `fix: resolve Y` — bug fix
   - `docs: update Z` — documentation only
   - `chore: bump deps` — maintenance
5. **Open a PR** against `main` with a clear description and test plan
6. We'll review within **72 hours**

## Ground Rules

- Don't modify smart contract ABIs without a corresponding Foundry test
- Don't add new npm packages without discussion in the PR
- All API routes must handle errors and not expose secrets in responses
- Keep PRs focused — one feature or fix per PR
